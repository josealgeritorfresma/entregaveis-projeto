# Branching e Versionamento

## Branching (sugestão)
- `main`: produção
- `develop`: integração (opcional)
- `feature/<ticket>`: novas features
- `fix/<ticket>`: correções
- `release/<versao>`: preparação de release (opcional)

## Versionamento
- SemVer: MAJOR.MINOR.PATCH
- Tags: `vX.Y.Z`

## Convenção de commits
Conventional Commits:
- feat:
- fix:
- chore:
- docs:
- test:
- refactor:
