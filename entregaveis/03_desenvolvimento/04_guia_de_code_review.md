# Guia de Code Review

## O que avaliar
- Correção funcional
- Segurança (validação, autorização, exposição de dados)
- Performance (queries, renderizações)
- Qualidade/legibilidade
- Testes (cobertura do comportamento)
- Observabilidade (logs e erros)

## Checklist de PR
- [ ] Passou lint/test/build
- [ ] Sem código morto / comentários desnecessários
- [ ] Tipagem correta (TS)
- [ ] Tratamento de erro consistente
- [ ] Acessibilidade (no frontend) revisada quando aplicável
